# Pre-Registration: Pharos v0.1.1 — β Calibration + X-ray HOT DOG

**Status:** FROZEN 2026-05-12.
**Supersedes:** [`v0.1_ir_benchmark.md`](v0.1_ir_benchmark.md) for §5 (confounder vector) only.
**Freeze record:** see [`FROZEN.md`](FROZEN.md) for the SHA-256 of this document at freeze time. Every other section of v0.1 — coverage class, stellar locus model, score calibration, quiet-negative population definition, follow-up class thresholds, pass criteria for the seven Hephaistos candidates — remains in force unchanged.

This document pre-registers two substantive changes to the v0.1 confounder model:

1. **β-weight calibration.** v0.1's β weights were pre-registered initialization values. v0.1.1 replaces them with values fit by a regularized logistic regression on a labeled `contaminant_positive` vs `quiet_negative` set.

2. **HOT DOG component redefinition.** v0.1's HOT DOG indicator was a 3-feature heuristic (faint-G + bright-W3/W4 + low-DSC-QSO). v0.1.1 adds an X-ray null requirement: a positional cross-match against public X-ray catalogs. A "HOT DOG" must have **no** X-ray detection within the search radius; an X-ray detection at the source position fails the HOT DOG criterion.

After freeze, the calibration is run **once** against the frozen contaminant set. No iteration, no "tune until Hephaistos passes." The Hephaistos benchmark is a held-out test of the calibrated weights.

## 1. Contaminant-Positive Control Set

The contaminant-positive set is fixed at freeze time. Composition:

### 1.1 Sub-populations

| Sub-population | Source catalog | Target count | Selection criteria |
|---|---|---|---|
| Confirmed AGN behind M-dwarfs | Hephaistos registry (G; A and B if `contaminant_suspected`) | 3 | `controls/hephaistos.yaml` status ≠ `unresolved_stress_test` |
| Million Quasars | Million Quasars Catalogue (Flesch 2023, v8.0) | ~100 | within v0.1 coverage class (parallax > 0 is rare for QSOs; use bright-flag QSOs that *do* have a Gaia DR3 match within 1.5″) |
| YSOs | Marton et al. 2019 (Gaia DR2 YSO probability ≥ 0.8) | ~100 | crossmatched to Gaia DR3 via `gaiadr2_neighbourhood`; require AllWISE+2MASS coverage |
| M-dwarf debris disks | Chen et al. 2014 + Theissen et al. additions | ~50 | confirmed disk status; M-dwarf primary |

Target total: ~250 contaminants. **Final count after applying the WISE-only coverage class filter (pre-reg v0.1 §2) is fixed at freeze and recorded in `controls/contaminant_positive/manifest.yaml`.**

### 1.2 Inclusion / exclusion rules

A source enters the contaminant set iff:
- It satisfies the v0.1 WISE-only coverage class (parallax_over_error ≥ 5, G ≤ 18, distance ≤ 200 pc, AllWISE crossmatch ≤ 3″, W1/W2 SNR ≥ 5, W3 SNR ≥ 3).
- It has a 2MASS counterpart within 1.5″.
- It is **not** in the quiet-negative control population (a source cannot be in both).

The 250-source target may not be achievable for all sub-populations within the 200 pc / G ≤ 18 cut. If a sub-population yields fewer than 30 sources after filtering, it is included as-is and the manifest records the actual count.

## 2. X-ray Cross-Match for HOT DOG

### 2.1 Catalogs

The X-ray-null check unions detections across the following public catalogs:

| Catalog | Reference | Sky coverage | Notes |
|---|---|---|---|
| 2RXS | Boller et al. 2016 (ROSAT All-Sky 2nd revision) | All-sky | shallow; ~135k sources |
| XMM-Newton DR12 | XMM-SSC 2022 | Pointed (~3% of sky cumulative) | deeper than ROSAT where covered |
| eROSITA-DE DR1 | Merloni et al. 2024 | German half-sky (galactic longitude 180°–360°) | deepest all-sky catalog available publicly |
| Chandra Source Catalog v2.0 | Evans et al. 2010+ | Pointed (~1% of sky) | deepest detections where covered |

Source positions are queried for each catalog independently via Vizier (or the catalog's primary archive). A source is "X-ray detected" iff **any** of the four catalogs has a counterpart within the search radius.

### 2.2 Search radius and match criterion

Crossmatch radius: **10 arcseconds**. This is conservative for the shallow surveys (2RXS positional uncertainty ~30″ for faint sources; we accept a tighter cut to avoid chance alignments) and generous for the deep ones (Chandra positional uncertainty < 1″).

A match is recorded iff a catalog source falls within 10″ of the Gaia DR3 position. The match count and minimum angular distance per catalog are stored in the join dataframe as:

| Column | Type | Description |
|---|---|---|
| `xray_2rxs_match_count` | int | sources within 10″ in 2RXS |
| `xray_2rxs_min_arcsec` | float | nearest 2RXS match angular distance |
| `xray_xmm_match_count` | int | …XMM-DR12 |
| `xray_xmm_min_arcsec` | float | …XMM-DR12 |
| `xray_erosita_match_count` | int | …eROSITA-DE DR1 |
| `xray_erosita_min_arcsec` | float | …eROSITA-DE DR1 |
| `xray_chandra_match_count` | int | …Chandra CSC v2 |
| `xray_chandra_min_arcsec` | float | …Chandra CSC v2 |
| `xray_any_detection` | bool | OR of the four `*_match_count > 0` |
| `xray_coverage_count` | int | number of catalogs whose footprint covers the source position |

`xray_coverage_count` records how many of the four catalogs **could have** detected the source (i.e., the position falls inside the catalog's sky footprint). A source with `xray_coverage_count == 0` is treated as "X-ray unconstrained" — the HOT DOG component is not modified by X-ray data for that source (defaults to the v0.1 heuristic).

### 2.3 Revised HOT DOG component definition

```
P_HOT_DOG_v0.1.1 = base_heuristic × (1 - X-ray_attenuation)
```

where:

- `base_heuristic` is the v0.1 indicator (faint-G ≥ 17.0 + bright-W3 ≤ 12.0 OR bright-W4 ≤ 9.0 + low-DSC-QSO < 0.5), unchanged.
- `X-ray_attenuation = 1.0` if `xray_any_detection AND xray_coverage_count > 0`, else `0.0`.

So:
- Source faint-G + bright-W3 + X-ray null → `P_HOT_DOG = 1.0` (strong contamination signal).
- Source faint-G + bright-W3 + X-ray detected → `P_HOT_DOG = 0.0` (X-ray evidence rules out HOT DOG even though the heuristic fires).
- Source faint-G + bright-W3 + X-ray unconstrained → `P_HOT_DOG = 1.0` (defaults to v0.1 heuristic; absence of coverage is not evidence).

## 3. β Calibration Procedure

### 3.1 Method

**Logistic regression with non-negative coefficient constraint and L2 regularization.** Implementation via scikit-learn's `LogisticRegression(penalty='l2', solver='lbfgs', max_iter=1000)` wrapped to apply non-negativity by reparameterizing β = exp(γ) and fitting γ unconstrained, OR by post-clipping (clip negatives to zero and refitting once).

Specifics:
- **Features:** the seven P_IR components (`P_blend`, `P_galaxy`, `P_qso`, `P_lowgalb`, `P_HOT_DOG_v0.1.1`, `P_badflag`, `P_NSS`) plus `P_wise_offset`. Eight features total.
- **Label:** binary, `1 = contaminant_positive`, `0 = quiet_negative`.
- **Class balance:** use `class_weight='balanced'` to compensate for class imbalance (~250 contaminants vs ~2451 quiet-negatives).
- **L2 strength:** `C = 1.0` (sklearn convention; inverse of regularization strength). Pre-registered; no tuning.
- **Non-negativity:** enforced via reparameterization β = max(0, γ_raw) after fit. Coefficients that come back negative after the L2 fit are clipped to zero and the loss is re-evaluated; this is the final calibrated β.
- **Bias term:** dropped (we want pure scale on the components, not an absolute offset).

### 3.2 Train / held-out split

The contaminant-positive set is **not** split. v0.1 already holds out the Hephaistos seven as the benchmark test; v0.1.1 keeps that. The contaminant-positive set is the fit set; Hephaistos is the test set.

This means:
- **If candidate G is in the contaminant_positive set, it must not also be used to define the v0.1.1 pass criterion.** The pre-reg keeps the v0.1 Hephaistos classification thresholds as the test; the v0.1.1 calibration must produce β weights such that the v0.1 Hephaistos thresholds still pass.

The calibration is therefore **never aware of the Hephaistos pass criterion**. It only sees the P_IR component vectors and the binary label of the *contaminant_positive* sub-populations.

### 3.3 Reproducibility

The calibration is deterministic given:
- The frozen contaminant set (recorded in `controls/contaminant_positive/manifest.yaml` and the resulting parquet at `controls/contaminant_positive/contaminants.parquet`).
- The frozen quiet-negative set (`controls/cache/quiet_negative.parquet`, hash recorded).
- The frozen X-ray cross-match table (`controls/cache/xray_crossmatch.parquet`, hash recorded).
- A fixed random seed (`random_state=0`).

A `controls/calibrated_betas_v0.1.1.yaml` artifact records the resulting β weights, the SHA-256 of each input fixture, and the scikit-learn version used.

## 4. Pass Criteria for v0.1.1

v0.1.1 passes iff **all** of the following hold:

1. **The v0.1 primary Hephaistos benchmark still passes** with the calibrated β weights and the X-ray-aware HOT DOG component. All seven candidates classified per v0.1 §7.

2. **The v0.1 secondary injection recovery still passes** (monotonic in σ; ≥ 90% recovery at 20σ).

3. **The v0.1 secondary FDR null-sample report still passes** (rejection count ≤ nominal q × N at q = 0.05, ideally zero rejections under the conservative-BH null behavior we observed in v0.1).

4. **The X-ray-null HOT DOG firing rate on the quiet-negative population is < 5%.** A clean control population should rarely trigger HOT DOG; a higher rate indicates the heuristic is mis-tuned.

5. **At least one calibrated β coefficient is non-zero.** A trivial all-zero solution is rejected.

If any of these fail, v0.1.1 does not ship; the failure is recorded and the contaminant-positive set or X-ray catalog selection may be revisited (which requires a v0.1.2 pre-reg document, **not** an edit to this one).
