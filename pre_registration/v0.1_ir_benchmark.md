# Pre-Registration: Pharos v0.1 — IR / Astrometric Confounder Benchmark

**Status:** Draft (unfrozen).
**Date opened:** 2026-05-10.
**Freeze required before:** any Gaia / AllWISE query touches the Hephaistos candidate neighborhoods or generates IR evidence scores against the wider sky.

This document fixes the scoring rules, control populations, and pass criteria for Pharos v0.1 *before* any candidate data is queried. After freeze, any change to this document requires a versioned superseding pre-registration document (`v0.1.1_*`) and re-running the benchmark from scratch. The benchmark is hygienic only to the extent that this document is frozen before contact with the candidate data.

---

## 1. Scope

Pharos v0.1 covers exactly the IR + astrometric confounder layer described in manuscript §3.1 and §3.2. No time-domain, spectral, optical pulse, radio, exoplanet architecture, or geometry-prior modalities are scored in v0.1. The output is `ir_evidence` and the IR-relevant confounder vector components, calibrated to empirical p-values within the WISE-only coverage class and FDR-controlled via Benjamini-Hochberg.

## 2. Stellar Population and Coverage Class

**WISE-only coverage class for v0.1** is defined as Gaia DR3 sources satisfying *all* of:

- `parallax_over_error >= 5`
- `phot_g_mean_mag <= 18.0`
- Distance (`distance_parallax_pc = 1000 / parallax_mas`) within 200 pc
- Successful match in `gaiadr3.allwise_best_neighbour` with `angular_distance <= 3.0″`
- AllWISE photometry exists in W1 and W2 with `w1snr >= 5` and `w2snr >= 5`
- AllWISE W3 photometry exists with `w3snr >= 3` (the residual channel; W4 is optional, scored only when `w4snr >= 2`)

Sources outside this coverage class are not scored in v0.1.

## 3. Stellar Locus Residual Model

For each in-class source, compute the color indices relative to the 2MASS Ks anchor:

- `w3_ks = w3mpro − tmass_ks_m`
- `w4_ks = w4mpro − tmass_ks_m` (computed only when W4 SNR ≥ 2)

The "stellar locus" for the IR layer is the median + interquartile range (IQR) of these color indices in the quiet-negative control population (Section 6), within each stratification cell (Section 4). The raw band-level residual for source *k* in band *b* is the robust z-score

```
s_{b,k} = (color_{b,k} − median_{b,bin(k)}) / (IQR_{b,bin(k)} / 1.349)
```

where `IQR / 1.349` is the Gaussian-equivalent σ from the control IQR. The 2MASS Ks anchor is required (sources lacking a 2MASS counterpart at angular distance ≤ 1.5″ are excluded from v0.1).

This is a data-driven replacement for the BT-Settl synthetic-grid approach used in some published IR-excess searches. The locus approach is internally consistent because the control population is itself defined as clean main-sequence sources (Section 6); the resulting locus is what a clean photosphere looks like in our data, without dependence on an external grid version. v0.1.1 may add a synthetic-grid cross-check as a secondary residual channel.

## 4. Calibrated IR Evidence

Per manuscript §5.1, for each band b:

```
s_b   = standardized W3 (or W4) magnitude excess
p_b   = (1 + #{c in C_IR : s_c >= s_b}) / (1 + |C_IR|)
E_b   = -log10(p_b)
```

where `C_IR` is the matched quiet-negative control population (Section 6 below), stratified by:

- Stellar type bin (effective temperature: <4000K, 4000–5500K, 5500–7000K, >7000K)
- G-magnitude bin (0.5 mag width)
- Distance bin (25 pc width)
- Galactic latitude bin (|b|: <10°, 10–30°, >30°)

The composite IR evidence is `ir_evidence = max(E_W3, E_W4)`. The choice of `max` is pre-registered to avoid post-hoc selection of the more favorable band.

## 5. Confounder Vector for IR Layer

`P_IR` contains the following components, each in [0, 1]:

| Component | Source | Initial β weight |
|---|---|---|
| `P_blend` | `1 - exp(-angular_distance / 1.5″)` × indicator(`number_of_neighbours > 1` or `number_of_mates > 0`) | 2.0 |
| `P_galaxy` | `classprob_dsc_combmod_galaxy` | 3.0 |
| `P_QSO` | `classprob_dsc_combmod_quasar` | 3.0 |
| `P_lowgalb` | `1` if `|b| < 10°` else `max(0, (20 - |b|) / 10)` | 0.5 |
| `P_HOT_DOG` | Heuristic: faint Gaia G + bright W3/W4 + no Gaia DSC quasar match | 2.5 |
| `P_badflag` | indicator(`cc_flags` contains D/H/O/P in W3 or W4; or `ext_flg > 0`; or `nb > 1`) | 2.0 |
| `P_NSS` | indicator(NSS table membership) | 1.0 |
| `P_wise_offset` | `min(1, √(Δα² + Δδ²) / 6.0″)` where (Δα, Δδ) is the W1↔W3 photocentre offset in arcsec | 5.0 |

Penalty is `β · P_IR` (linear). β weights are pre-registered initialization values; calibration against the contaminant-positive control set is performed once after v0.1 ships and reported as a v0.1.1 update.

**On `P_wise_offset`.** A significant offset between the W1 (3.4 μm) and W3 (12 μm) photocentres is a direct indicator of WISE-internal blending: a single physical source has a single photocentre across all WISE bands (within the per-band PSF FWHMs). The 6.0″ scale is set so that the published Theissen & West (2017) quasar offset distribution (μ≈−0.21″, σ≈5.48″) maps to roughly P=0.9 at a ~1σ outlier. The β weight of 5.0 is the highest in the vector because offset is the most diagnostic IR-only contamination signal — a Suazo et al. (2024) candidate flagged on this signal (candidate G, offset 5.59″) was subsequently confirmed by high-resolution e-MERLIN/EVN observations to be background AGN contamination.

For v0.1, photocentre offsets are read from a pre-measured input column (`w1_w3_offset_arcsec_ra`, `w1_w3_offset_arcsec_dec`) when present. For the Hephaistos benchmark these values are loaded from Suazo et al. (2024) Table 7. For the general quiet-negative control population they are missing and default to 0, which is appropriate because the control population is independently selected to exclude WISE-confused sources (Section 6: `number_of_neighbours == 1`, `number_of_mates == 0`, angular distance ≤ 1.5″). A general-population unWISE re-extraction pipeline (Suazo et al. method, SEP on unWISE coadds) is deferred to v0.2 of the IR layer.

## 6. Quiet-Negative Control Population (`C_IR`)

`C_IR` is the population against which empirical p-values are computed. The pre-registered definition is *all Gaia DR3 sources satisfying* the WISE-only coverage class (Section 2) *and additionally*:

- `ruwe < 1.2`
- `phot_bp_rp_excess_factor` within ±0.1 of the magnitude-binned median (Riello et al. 2021 prescription)
- Not in NSS tables
- `classprob_dsc_combmod_galaxy < 0.1` and `classprob_dsc_combmod_quasar < 0.1`
- AllWISE crossmatch has `number_of_neighbours == 1` and `number_of_mates == 0`
- `angular_distance < 1.5″` in the AllWISE best-neighbour join
- `|b| > 20°` (Galactic plane and disk excluded from the control)
- No `cc_flags` contamination indicators in W1, W2, W3
- No cross-match within 30″ to (a) SIMBAD-known YSO classifications, (b) SIMBAD-known AGN, (c) Chen et al. debris disk catalogs

Control population is built via a single Gaia archive TAP query (query text pinned in the freeze). Target size is `|C_IR| ≥ 50,000` after stratification.

## 7. Pass Criteria

For each of the seven Hephaistos candidates (Suazo et al. 2024), Pharos v0.1 passes iff:

| Candidate | Required follow-up class | Confounder signal |
|---|---|---|
| G (confirmed background AGN, MNRAS Letters high-resolution follow-up) | `discard_confounded` | W1↔W3 photocentre offset 5.59″ |
| B | `discard_confounded` *or* `needs_human_review` | W1↔W3 photocentre offset 3.21″ |
| A | Not in `ir_followup` | (Demotion to `discard_confounded` / `needs_human_review` requires the radio archival cross-match modality from v0.3; not feasible with v0.1 IR-only data. Pass criterion for v0.1 is that A is not auto-promoted to top-tier IR follow-up.) |
| Other 4 (unresolved: C, D, E, F) | Not in `ir_followup` *unless* manual inspection of WISE centroid, crossmatch quality, local neighborhood, and SED morphology survives |  |

Additionally:
- Empirical FDR (`fdr_q_value_within_coverage_class`) for the v0.1 ranked list is reported at q ∈ {0.01, 0.05, 0.1} thresholds.
- A `synthetic_injection` set of at least 100 injected IR-excess sources (controlled W3 excess at 5σ, 10σ, 20σ; varied stellar types, distances, latitudes) is recovered at expected per-σ recovery rate.

## 8. Follow-Up Class Thresholds

Within the WISE-only coverage class:

```
LeadScore_v0.1 = ir_evidence - β · P_IR - D
```

with `D` being the data quality penalty from `ruwe`, `astrometric_excess_noise_sig`, and `phot_bp_rp_excess_factor`.

Class assignment (cuts pre-registered, exposed in the output schema):

| Class | Condition |
|---|---|
| `discard_confounded` | `β · P_IR ≥ 4.0` |
| `needs_human_review` | `2.0 ≤ β · P_IR < 4.0` *and* `ir_evidence ≥ 2.0` |
| `ir_followup` | `LeadScore_v0.1 ≥ 3.0` *and* `β · P_IR < 2.0` *and* `fdr_q_value_within_coverage_class ≤ 0.05` |
| `no_action` | Otherwise |

## 9. Public Release Artifacts (v0.1)

Per manuscript §8, v0.1 releases:

- `lead_graph_catalog_v0.1.csv` — ranked sources in the WISE-only coverage class
- `modality_scores.parquet` — raw and calibrated E_IR per source
- `confounder_vectors.parquet` — `P_IR` components per source
- `coverage_masks.parquet` — `q_IR` per source
- `fdr_by_coverage_class.csv` — empirical p-values and BH q-values
- `controls/` — full quiet-negative, contaminant-confirmed, contaminant-suspected, synthetic-injection sets
- This document (frozen, hash recorded in the release notes)
- Reproduction notebook for the Hephaistos pass criterion

## 10. What v0.1 Explicitly Does *Not* Claim

- v0.1 makes no technosignature detection claim.
- v0.1 makes no statement about sources outside the WISE-only coverage class.
- v0.1 does not score time-domain, spectral, radio, optical pulse, or geometric channels; these are deferred to later versions.
- v0.1's β weights are initialization values, not calibrated weights — calibration against the contaminant-positive set ships in v0.1.1.
- A source landing in `no_action` is not a "negative result" — it means insufficient cross-modal evidence in the only modality v0.1 measures.
