## What ran

The pipeline executed end-to-end against the live Gaia archive on 2026-05-12:

1. **Contaminant-positive set built** per pre-reg §1:
   - Hephaistos labeled contaminants (G, A, B) from `controls/hephaistos.yaml`.
   - Marton et al. 2019 YSOs (`II/360/catalog`), filtered SY ≥ 0.8, capped at 5000 catalog rows. CDS XMatch returned 4,998 Gaia DR3 matches.
   - Chen et al. 2014 debris disks (`J/ApJS/211/25`). CDS XMatch returned 438 Gaia DR3 matches.
   - Million Quasars skipped (parallax-positive QSOs are essentially absent from the v0.1 coverage class).

2. **Gaia + AllWISE + 2MASS join fetched** via the 3-step split (gaia_source, AllWISE join, 2MASS join), 11 chunks of 500 IDs each. (The original 5-way single-query JOIN repeatedly returned phase=ERROR on the Gaia archive during the DR4 transition; the split path works.)

3. **Coverage filter applied** per v0.1 §2: 5,436 → 518 sources after parallax_over_error / G-mag / distance / AllWISE-crossmatch / 2MASS-crossmatch cuts.

4. **Quiet-negative exclusion applied** per pre-reg §1.2: 518 → 495 sources after removing 23 that also appear in `controls/cache/quiet_negative.parquet`.

5. **Calibration ran** per pre-reg §3 (non-negative L2 logistic regression, C=1.0, no intercept, class_weight='balanced', random_state=0). Result archived at [`v0.1.1_attempt1_calibrated_betas.yaml`](v0.1.1_attempt1_calibrated_betas.yaml).

## Why it failed

**Pre-reg §4.1: Hephaistos benchmark must still pass.** With the calibrated β, the v0.1.0 Hephaistos benchmark fails on candidates G and B:

| Candidate | v0.1.0 class | v0.1.1 attempt-1 class | Required |
|---|---|---|---|
| G | discard_confounded ✓ | **no_action** ✗ | discard_confounded |
| B | discard_confounded ✓ | **no_action** ✗ | discard or NHR |
| A | no_action ✓ | no_action ✓ | not ir_followup |
| C, D, E, F | no_action ✓ | no_action ✓ | not ir_followup |

G and B were both demoted in v0.1.0 specifically via the `wise_offset` component (W1↔W3 photocentre offset 5.59″ for G, 3.21″ for B). The calibrated β for `wise_offset` is **0.056** — effectively zero — so the offset signal no longer contributes meaningfully to either candidate's confounder penalty.

## Why the calibration crushed wise_offset

The 495-source contaminant set is dominated by **two sub-populations that don't have W1↔W3 photocentre offsets**:

- 295 Chen debris disks: real point sources with circumstellar dust IR excess. The IR excess is co-located with the stellar photocentre; no WISE photocentre offset.
- 199 Marton YSOs: similar — accreting young stars with disk IR excess at the stellar position.

Only the 1 Hephaistos contaminant (candidate A) is in the calibration set, and A's W1↔W3 offset is just 0.33″ — well below the scale on which the component fires.

The logistic regression therefore sees `wise_offset` firing for ~0% of contaminants → concludes the feature is uninformative → drives its β toward zero (regularized to 0.056). The bulk of the contaminant set teaches the model to weight features that ARE common across YSOs and debris disks: `low_galactic_lat` (both populations cluster near the Galactic plane) and `bad_flag` and `nss` (variability flags fire more often in young / disk-bearing systems).

## What v0.1.2 needs

To rescue calibration of `wise_offset`, the v0.1.2 pre-registration must add at least one sub-population that **has** WISE photocentre offsets:

**Option A (recommended):** synthetic-offset positives. Take ~50 quiet-negative control sources, inject controlled W1↔W3 photocentre offsets at e.g. 3″, 5″, 8″, and add them to the contaminant set with a synthetic-offset label. This teaches the calibration that the feature is real even though catalog sources rarely have it.

**Option B:** mine an existing catalog of confirmed WISE-blend or background-AGN cases. Ren, Garrett & Siemion 2024 cross-matched Hephaistos candidates with archival radio sources and named candidates A, B, G as offset-suspects — possibly a small but high-quality "wise_offset positive" sub-population.

**Option C:** stratified calibration — fit β separately per sub-population (debris/YSO/offset/etc.), combine. More complex; defers to v0.2 alongside the time-domain layer.

The v0.1.2 pre-reg should also acknowledge that the current logistic-regression form averages signal across heterogeneous contaminant types, and document this as a model limitation regardless of which Option above is chosen.

## What's preserved from this attempt

The infrastructure built for v0.1.1 attempt-1 is functional and remains in the repository:

- `src/pharos/xray.py` — X-ray cross-match module (works against Vizier).
- `src/pharos/contaminants.py` — contaminant-positive set fetcher (CDS XMatch + 3-step Gaia split).
- `src/pharos/calibration.py` — non-negative L2 logistic regression (verified by unit tests).
- `pharos.cli` commands: `fetch_xray`, `fetch_contaminants`, `run_calibration`.
- Confounders module supports loading calibrated β from a YAML artifact when present; the v0.1.1 auto-load path looks at `controls/calibrated_betas_v0.1.1.yaml`.

Re-running `pharos.cli run_calibration` against an expanded v0.1.2 contaminant set should yield β values that satisfy the Hephaistos pass criterion.
